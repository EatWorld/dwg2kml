# CGCS2000 与高斯克吕格投影参考

## 1. CGCS2000（2000国家大地坐标系）

- **全称**：China Geodetic Coordinate System 2000
- **类型**：地心大地坐标系
- **椭球参数**：长半轴 a = 6378137 m，扁率 f = 1/298.257222101
- **与 WGS84 差异**：厘米级，日常应用可视为等价
- **EPSG代码**：4490（地理坐标系）

## 2. 高斯克吕格投影

- **类型**：横轴墨卡托投影（Transverse Mercator）
- **特点**：等角投影，中央经线无变形，离中央经线越远变形越大
- **分带方式**：
  - **3度带**：从 1.5°E 开始，每 3° 一带，中央经线 = 3° × 带号
  - **6度带**：从 0°E 开始，每 6° 一带，中央经线 = 6° × 带号 - 3°
- **坐标偏移**：
  - **北坐标 X**：直接为距赤道距离（米），无假偏
  - **东坐标 Y**：含假东 500000m（避免负值），3度带常在 Y 前加带号前缀

## 3. Y值带号前缀识别

CGCS2000 高斯克吕格坐标的 Y 值（东坐标）常见两种格式：

| 格式 | 示例 | 含义 |
|------|------|------|
| 含带号 | `39530339.816` | 带号=39，实际东坐标=530339.816 |
| 不含带号 | `530339.816` | 实际东坐标=530339.816（需另知带号） |

**判别方法**：
- 8 位数（>1000000）→ 含带号，前 2 位为带号
- 6 位数（~500000 附近）→ 不含带号

## 5. 安徽蚌埠区域参数

- **带号**：39（3度带）
- **中央经线**：117°E
- **EPSG**：4548
- **大致经纬度范围**：北纬 32.7°-33.1°，东经 117.2°-117.5°
- **典型坐标值**：
  - 北坐标 X ≈ 3630000 - 3660000
  - 东坐标 Y ≈ 39500000 - 39600000（含带号 39）

## 6. 转换公式（pyproj 示例）

```python
from pyproj import Transformer

# 步骤1：根据带号选择EPSG（蚌埠用4548）
transformer = Transformer.from_crs("EPSG:4548", "EPSG:4326", always_xy=True)

# 步骤2：去掉Y值带号前缀
north_x = 3641349.088      # 北坐标（距赤道距离，米）
east_y_raw = 39530339.816  # 含带号的东坐标
zone = 39
true_easting = east_y_raw - zone * 1000000  # 530339.816

# 步骤3：转换（注意参数顺序：先东后北）
lon, lat = transformer.transform(true_easting, north_x)
# 结果：lon≈117.32°, lat≈32.90°
```

## 7. 常见错误排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 转换结果为 `inf` | Y值未去带号，超出投影范围 | `east_y -= zone * 1000000` |
| 位置偏几十公里 | EPSG选错带号（中央经线不对） | 检查Y前2位，按对照表选EPSG |
| 位置偏几百米~1公里 | CGCS2000 vs 北京54/西安80 | 确认源坐标系，换对应EPSG |
| 整体旋转偏移 | 投影分带类型错误（3度 vs 6度） | 确认图纸标注的分带类型 |
| EPSG偏移约300km | 误用6度带EPSG（如4559） | 3度带公式: EPSG = 4509 + 带号 |

## 8. EPSG 计算公式

**3度带**: `EPSG = 4509 + 带号`
- 验证: 39 → 4509+39 = 4548 ✓ (CGCS2000 / 3-degree Gauss-Kruger CM 117E)

**6度带**: `EPSG = 4491 + 带号`（不推荐，征迁图通常用3度带）

⚠️ 常见错误: 用 `4520 + 带号` 得到的是6度带代码（如 39→4559），会偏移约300km。

## 9. KML 颜色格式速查

KML `<color>` 标签格式为 **AABBGGRR**（alpha-blue-green-red），与常规 RGB 相反：

| 颜色 | 代码 | 说明 |
|------|------|------|
| 不透明红色 | `ff0000ff` | AA=ff, BB=00, GG=00, RR=ff |
| **不填充(仅边框)** | `fill=0` | `<fill>0</fill>` 无内部颜色 |
| 不透明蓝色 | `ffff0000` | AA=ff, BB=ff, GG=00, RR=00 |
| 不透明绿色 | `ff00ff00` | AA=ff, BB=00, GG=ff, RR=00 |
| 不透明黑色 | `ff000000` | 全0 |

⚠️ 常见错误: 写 `ffff0000` 以为是红色，实际是蓝色！红色应为 `ff0000ff`。
